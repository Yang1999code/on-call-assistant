### 发现的问题与修复
1. **FTS5 contentless TEXT id 不兼容**：TEXT 主键无法用于 content_rowid（仅支持 INTEGER），改为 stored-content 模式
2. **snippet() 返回分词文本**：FTS5 snippet() 从 tokenized 文本生成，显示"服务 大面积 超时"，改为手动从 documents 表生成 snippet
3. **2字中文词无 trigram 匹配**："故障"（2字符）无法生成 3-gram，增加 `< 3` 字符 LIKE 回退分支
4. **Windows curl GBK 编码**：curl 在中文 Windows 上默认用 GBK 编码非 ASCII 字符，导致中文查询返回空。用 Python urllib 替换验证，API 本身正确处理 UTF-8

### 2026-05-15 全面审计与修复

基于 `question-1/README.md` 13 个验证用例进行全面审计，发现并修复 3 个问题。

#### 问题 1：V2-C1 语义搜索回归（"服务器挂了"找不到 sop-001）

- **现象**：`/v2/search?q=服务器挂了` 返回 7 个结果，sop-001 完全缺失
- **根因①**：ChromaDB 持久化数据 (`data/chroma/`) 中的向量嵌入与当前 `encode()` 输出不匹配 — 直接用 `encode()` 计算相似度得分 0.27-0.49，但 ChromaDB 查询返回 0.016-0.032，说明 collection 内嵌是旧版/不同模型生成的
- **根因②**：`data/search.db` 被之前测试污染 — `sop-001` 标题变成 "Updated"（V1-F7 测试后残留），`test-debug`、`test-interview`、`test-verify` 等测试文档未清理
- **修复**：删除 `data/chroma/` 和 `data/search.db`，重启服务器触发 lifespan 从头重新索引（从原始 10 份 HTML）
- **结果**：V2-C1 恢复通过，"服务器挂了" 正确返回 sop-001（rank 4）+ sop-004（rank 8）

#### 问题 2：Phase 1 router.py 的 ChromaDB 同步导致数据污染

- **现象**：Phase 1 的 POST /v1/documents 同时写入 FTS5 和 ChromaDB。V1-F7 测试用 `{title: "Updated", content: "Updated"}` 重索引 sop-001，导致 ChromaDB 中 sop-001 的嵌入变成 "Updated content" 的向量，后续 V2-C1 语义搜索再也找不到原始"后端服务"内容
- **根因**：Phase 1 和 Phase 2 关注点混淆 — Phase 1 负责关键词搜索（FTS5），不应管理 Phase 2 的向量索引
- **修复**：从 Phase 1 POST handler 中移除 ChromaDB 同步调用。Phase 2 的向量索引仅在服务器启动时通过 `index_all()` 从 FTS5 全量构建，保持关注点分离
- **附带清理**：移除 router.py 中未使用的 `_get_vector_store()` 函数和 `_vector_store` 全局变量（上一轮的 lazy singleton 残骸，定义但从未正确调用）

#### 问题 3：router.py lazy singleton 未正确连线

- **现象**：上一轮定义了 `_get_vector_store()` 函数，但 POST handler 中仍引用不存在的 `vector_store` 变量（NameError）。因服务器进程是旧代码（.pyc 时间戳 13:07，源文件时间戳 13:47），旧代码仍在模块级 `vector_store = VectorStore()` 下运行，实际未触发此 bug
- **修复**：清理死代码，Phase 1 router 回归纯粹 — 只依赖 SearchEngine（FTS5），不涉及 VectorStore

- **问题**：V1-F7 测试将 `sop-001` 重索引为 `{title: "Updated", content: "Updated"}` 后，数据残留到 `search.db`。用户手动双击 `start.bat` 时看到被污染的搜索结果（sop-001 显示 "Updated"），体验很差

  - **3 条 Hard Gates**（条件不满足不能继续）：readFile 返回内容前不能组织回答等
  - **Response Rules**：结构化回答（原因→排查→缓解→升级）、中文优先
